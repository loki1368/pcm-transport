What it is

PCM Transport is a compact Linux desktop audio player focused on direct PCM playback, transparent signal-path reporting, native FLAC decoding and optional DSP processing.

Native FLAC playback

Uses libFLAC for native FLAC decoding and FFmpeg / FFprobe for additional formats.

Additional formats via FFmpeg

Uses FFmpeg and FFprobe at runtime for MP3/MP2, M4A/M4R, AAC, AC-3, DTS, OGG/OPUS/Speex, WAV/W64, AIFF, APE, WV and selected legacy formats.

DSD to PCM conversion

Detects DSF and DFF sources, including DST-compressed DFF, through FFprobe and converts them through the existing FFmpeg and SoXr path to configurable PCM rates and 16/24/32-bit output.

Direct ALSA output

Designed for Linux playback with GTK 3 and ALSA, without requiring a mandatory PulseAudio or PipeWire layer.

MPRIS desktop integration

Desktop media controls, multimedia keys and cover art metadata. Initial implementation: loki1368.

Linux desktop integration

Embedded GTK window icons plus a desktop entry and hicolor icon set for optional system installation.

Gapless-oriented playback

Continuous CUE image playback, local M3U/M3U8 import and same-format file chaining are used where possible. Legacy Windows-1251 text is normalized to UTF-8.

Transparent signal path

Shows the active playback chain, format and processing state.

DSP Studio

Includes bass/treble, Deep Bass, soft volume, Pre-EQ Headroom and optional processing rules.

Diagnostics and indicators

Provides level metering, clip detection, active ALSA output reporting, ALSA device probing and bit-perfect testing.

Why another Linux audio player?

PCM Transport is built for users who want a small Linux FLAC player with a visible signal path, native libFLAC decoding, direct ALSA output, CUE support, gapless-oriented playback and optional DSP that can be fully bypassed.

The goal is not to replace large music library managers. PCM Transport is closer to a focused playback tool for local lossless audio: simple controls, honest runtime reporting and a clear distinction between native playback, conversion paths and optional processing.

It may be useful for Linux users interested in a lightweight open-source audio player, an ALSA-based FLAC player, a GTK desktop player, or a compact tool for experimenting with transparent DSP and playback diagnostics.

Supported formats

Native FLAC playback is handled through libFLAC. Additional formats use FFmpeg and FFprobe at runtime. DSF and DFF sources, including DST-compressed DFF, are converted to PCM; native DSD and DoP output are not used. W64, VOC and M4R are classified from the probed codec. Supported local audio paths may also be used by M3U/M3U8 and CUE. Rare-format support depends on the local FFmpeg build.

Native / primary

*.flac, *.wav, *.wave, *.cue, *.m3u, *.m3u8

FFmpeg-backed lossless / PCM / containers

*.aif, *.aiff, *.ape, *.wv, *.w64, *.bwf, *.au, *.snd, *.caf, *.voc, *.tak, *.tta, *.dsf, *.dff

FFmpeg-backed lossy

*.mp3, *.mp2, *.m4a, *.m4r, *.aac, *.ac3, *.dts, *.ogg, *.oga, *.opus, *.spx, *.ra, *.wma, *.asf, *.xwma, *.wmv, *.oma, *.aa3, *.at3, *.mpc, *.mp+, *.mpp

Download and run

The easiest way to try PCM Transport is the ready-to-run Linux x86_64 AppImage from GitHub Releases. Source build instructions are provided below for development and packaging.

AppImage Quick start 
chmod +x PCM-Transport-latest-x86_64.AppImage
./PCM-Transport-latest-x86_64.AppImage
Arch Linux Build dependencies 
sudo pacman -S --needed base-devel cmake pkgconf alsa-lib flac gtk3 ffmpeg
Debian / Ubuntu Build dependencies 
sudo apt install build-essential cmake pkg-config \
libasound2-dev libflac-dev libgtk-3-dev ffmpeg
ALT Linux Build dependencies 
su -
apt-get install build-essential cmake pkg-config libalsa-devel libflac-devel libgtk+3-devel ffmpeg
exit
Build Configure and compile 
cmake -S . -B build
cmake --build build -j
Run Start the player 
./build/pcm_transport
Install Optional desktop integration 
cmake --install build --prefix <prefix>

Installs the binary, desktop entry and hicolor icons. Running directly from build does not require installation.

Runtime dependencies include ALSA, GTK 3, libFLAC, FFmpeg and FFprobe. RTKit is optional for realtime audio-thread priority. pkexec/setcap is optional for persistent cap_sys_nice permission. Native FLAC playback can work without FFmpeg. FFmpeg/FFprobe are required for external formats, metadata probing and conversion paths. RTKit support uses GLib/GIO D-Bus from GTK 3 and does not add a separate build dependency. Persistent realtime permission uses pkexec/setcap cap_sys_nice and requires restart. Selected FFmpeg-backed formats include W64, MP2, M4R, AC3, DTS, Speex, VOC, RealAudio, AU/SND, BWF, CAF, TAK, TTA, WMA/WMV, ATRAC and MPC. DSF and DFF sources, including DST-compressed DFF, use configurable PCM conversion through FFmpeg and SoXr.

Project goals

The project is developed as a small, understandable and conservative Linux player: stable playback first, transparent signal path, predictable DSP behavior, ALSA diagnostics and no hidden processing.
